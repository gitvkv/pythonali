# Type Checking with the type() Function
<span class="category-tag">🏷️ Python Basics: Syntax, Variables, and Types /  Type Checking and Type Conversion</span>

=== "📖 Theory"
    When working with Python, it's common to need to know what kind of data you're dealing with. Whether you're debugging a script, validating input, or just getting familiar with how Python handles different values, knowing how to check a variable's type is a fundamental skill. Python makes this easy with the built-in **type()** function.
    
    ---
    
    ### 🕵️ What is the type() Function?
    
    The **type()** function returns the data type of any object or variable you pass to it. It tells you whether something is a string, integer, list, dictionary, or any other Python type.
    
    - Simply pass a value or variable inside the parentheses.
    - The function returns the type as a result (e.g., **<class 'int'>**, **<class 'str'>**).
    - You can use this output directly or store it for later comparison.
    
    ---
    
    ### ⚙️ Basic Usage Examples
    
    Here are some straightforward examples of using **type()** with common data types:
    
    - **type(42)** returns **<class 'int'>**
    - **type(3.14)** returns **<class 'float'>**
    - **type("Hello")** returns **<class 'str'>**
    - **type(True)** returns **<class 'bool'>**
    - **type([1, 2, 3])** returns **<class 'list'>**
    - **type({"key": "value"})** returns **<class 'dict'>**
    
    You can also check the type of a variable:
    
    - Assign **name = "Alice"** then call **type(name)** — this returns **<class 'str'>**
    - Assign **count = 10** then call **type(count)** — this returns **<class 'int'>**
    
    ---
    
    ### 📊 Comparing Common Data Types
    
    The table below shows some everyday Python values and their corresponding types when checked with **type()**:
    
    | Value | type() Output | Type Name |

    | "hello" | <class 'str'> | String |

    | {"a": 1} | <class 'dict'> | Dictionary |

    ---
    
    ### 🛠️ Practical Use Cases for Engineers
    
    Knowing how to check types is especially useful in real-world scenarios:
    
    - **Validating user input** — Confirm that a value entered by a user is a number before performing calculations.
    - **Debugging unexpected behavior** — When a function returns something unexpected, check its type to understand what went wrong.
    - **Writing safer code** — Use type checks to ensure your code handles different data formats correctly.
    - **Working with APIs or files** — Data from external sources can come in various types; checking them helps avoid errors.
    
    Example scenario: You receive a value from a configuration file and want to ensure it's an integer before using it in a loop. You can check **type(value)** to confirm it's **<class 'int'>**.
    
    ---
    
    ### 🔍 Checking Types Conditionally
    
    You can use **type()** inside conditional statements to make decisions based on data type:
    
    - If **type(data) == type("")** then treat it as a string.
    - If **type(data) == type([])** then treat it as a list.
    - If **type(data) == type({})** then treat it as a dictionary.
    
    This approach helps you write flexible code that adapts to different input types without crashing.
    
    ---
    
    ### 💡 Tips for Using type() Effectively
    
    - **Compare directly to type names** — Use **type(variable) == int** or **type(variable) == str** for clear, readable checks.
    - **Remember that type() returns a class** — The output is always in the format **<class 'typename'>**.
    - **Use type() for quick debugging** — When something doesn't work as expected, print the type of your variables to spot mismatches.
    - **Combine with print()** — A simple **print(type(my_var))** can save hours of troubleshooting.

=== "💻 Code"
    The `type()` function returns the data type of any Python object, allowing engineers to verify what kind of value they are working with.
    
    ---
    
    ### 🔧 Example 1: Checking the type of a simple integer
    
    This example shows how `type()` identifies whole numbers as integers.
    
    ```python
    value = 42
    result = type(value)
    print(result)
    ```
    
    📤 Output: **`<class 'int'>`**
    
    ---
    
    ### 🔧 Example 2: Checking the type of a decimal number
    
    This example demonstrates that numbers with decimal points are classified as floats.
    
    ```python
    value = 3.14
    result = type(value)
    print(result)
    ```
    
    📤 Output: **`<class 'float'>`**
    
    ---
    
    ### 🔧 Example 3: Checking the type of a text string
    
    This example shows how `type()` identifies text enclosed in quotes as a string.
    
    ```python
    value = "Hello, engineer"
    result = type(value)
    print(result)
    ```
    
    📤 Output: **`<class 'str'>`**
    
    ---
    
    ### 🔧 Example 4: Checking the type of a boolean value
    
    This example demonstrates that `True` and `False` are their own distinct data type.
    
    ```python
    value = True
    result = type(value)
    print(result)
    ```
    
    📤 Output: **`<class 'bool'>`**
    
    ---
    
    ### 🔧 Example 5: Using type() to compare two values before an operation
    
    This example shows a practical check engineers might use before performing addition.
    
    ```python
    first_value = 10
    second_value = "5"
    first_type = type(first_value)
    second_type = type(second_value)
    print(first_type)
    print(second_type)
    are_same = first_type == second_type
    print(are_same)
    ```
    
    📤 Output: **`<class 'int'>`**  
    📤 Output: **`<class 'str'>`**  
    📤 Output: **`False`**
